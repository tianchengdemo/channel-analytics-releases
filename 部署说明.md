# Channel Traffic Analytics 部署说明

## 平台对照

| 文件名 | 操作系统 | 架构 | 适用场景 |
|--------|---------|------|---------|
| site-analytics-linux-amd64 | Linux | x86_64 | 大多数云服务器 |
| site-analytics-linux-arm64 | Linux | ARM64 | 树莓派、ARM 服务器 |
| site-analytics-darwin-amd64 | macOS | Intel | Intel Mac |
| site-analytics-darwin-arm64 | macOS | Apple Silicon | M1/M2/M3 Mac |
| site-analytics-windows-amd64.exe | Windows | x86_64 | Windows 服务器 |

## 快速部署 (Linux)

### 1. 上传二进制和配置

```bash
# 选择对应架构的二进制（以 linux-amd64 为例）
scp release/v0.3.0/site-analytics-linux-amd64 user@server:/opt/site-analytics/
scp config.yaml user@server:/opt/site-analytics/
```

### 2. 服务器端配置

```bash
ssh user@server

cd /opt/site-analytics
chmod +x site-analytics-linux-amd64
mv site-analytics-linux-amd64 site-analytics

# 编辑配置文件（必须修改以下项）
vi config.yaml
```

**必须修改的配置项：**

```yaml
admin:
  username: "admin"
  password: "你的强密码"            # 启动时自动转 bcrypt 哈希
  token_secret: "随机字符串32位+"   # JWT 签名密钥

reporting:
  token: "随机字符串"               # 服务端上报认证令牌，不用可留空
```

### 3. 启动服务

```bash
# 前台运行（测试用）
./site-analytics -config config.yaml

# 后台运行
nohup ./site-analytics -config config.yaml > /dev/null 2>&1 &
```

### 4. 验证

```bash
# 检查服务
curl -s http://localhost:8080/admin/login.html | head -1

# 测试登录
curl -s http://localhost:8080/api/admin/login \
  -X POST -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"你的密码"}'
```

## systemd 部署 (推荐)

```ini
# /etc/systemd/system/site-analytics.service
[Unit]
Description=Channel Traffic Analytics
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/site-analytics
ExecStart=/opt/site-analytics/site-analytics -config /opt/site-analytics/config.yaml
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable site-analytics
sudo systemctl start site-analytics
sudo systemctl status site-analytics
```

## Nginx 反向代理 (推荐)

将统计服务挂在业务子域下，确保 Cookie 为 first-party：

```nginx
server {
    listen 80;
    server_name stats.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

配置 `config.yaml` 中 trusted_proxies：

```yaml
proxy:
  trusted_proxies:
    - "127.0.0.1/32"
```

## Docker 部署

```bash
# 使用项目根目录的 docker-compose.yml
docker-compose up -d

# 或手动运行
docker build -t site-analytics .
docker run -d \
  --name site-analytics \
  -p 8080:8080 \
  -v ./data:/app/data \
  -v ./config.yaml:/app/config.yaml:ro \
  site-analytics
```

## Windows 部署

```powershell
# 直接运行
.\site-analytics-windows-amd64.exe -config config.yaml

# 注册为 Windows 服务（使用 nssm）
nssm install SiteAnalytics "C:\site-analytics\site-analytics-windows-amd64.exe"
nssm set SiteAnalytics AppParameters "-config C:\site-analytics\config.yaml"
nssm set SiteAnalytics AppDirectory "C:\site-analytics"
nssm start SiteAnalytics
```

## macOS 部署

```bash
chmod +x site-analytics-darwin-arm64  # 或 darwin-amd64
./site-analytics-darwin-arm64 -config config.yaml
```

首次运行可能需要在"系统设置 > 隐私与安全性"中允许运行。

## 接入统计

服务启动后，登录后台 `http://your-domain:8080/admin/`，创建渠道后点击"获取代码"，复制对应的接入代码到业务页面：

```html
<!-- 异步 JS（推荐） -->
<script>
(function(){
  var d=document,s=d.createElement('script');
  s.src='//stats.yourdomain.com/sa.js?cc=渠道编码';
  s.async=true;
  (d.head||d.body).appendChild(s);
})();
</script>
```

## 数据备份

SQLite 数据库文件默认在 `./data/stats.db`：

```bash
# 备份
cp data/stats.db data/stats.db.bak.$(date +%Y%m%d)

# 或使用 sqlite3 在线备份
sqlite3 data/stats.db ".backup 'data/backup.db'"
```
